Chapter 1: Introduction to HTML

HTML is a markup language for describing web documents (web pages). It stands for Hyper Text Markup Language. A markup language is a set of markup tags. HTML documents are described by HTML tags. Each HTML tag describes different document content. An HTML tag is always contained within angled brackets. Most tags have an opening tag (<p> for example) and a closing tag, (</p>). An HTML element refers to everything within a set of opening and closing tags. There is also the term "attriute". This is a property of an HTML element. For example, to set the href attribute of an anchor tag to the Udacity URL, you would write <a href="www.udacity.com">

HTML examples

  • <b> = bold
  • <em> = emphasis/italic
  • <a href = "..."> = link f.ex. this is derp

There are different kind of html elements. Usually each html starting tag also has an end tag. But sometimes tags don't need to be closed. These kind of tags are called void tags. Void elements are a special kind of elements that must not have content.

Void elements

  • <br> = line break
  • <img src="url" alt="text"> = adding an image to the text

Other kind of html elements are defined as block level elements or inline elements. Block level elements create invisible boxes that shift to a new line (f.ex. h1, p, ul, div, form). Inline elements are normally displayed without line breaks (f.ex. br, a, img, span). The actual html content goes into the <body>-section. Chapter 2: Creating a Structured Document with HTML

Purposes of HTML, CSS and Javascript

A website is like a house. Html is its structure, its architect’s plan. CSS are then the style – colors, decoration and so on. Java Script files are interactive components like the electric garage door.
We also learn that not all elements of the source code are visible on the website but for sure all elements are ordered in rectangular boxes. The structure of all websites containing these boxed elements is tree-like.

Bullet points

  • HTML and CSS are both languages
  • HTML controls the structure of a web page
  • CSS controls the style of a page (how it looks)
  • When programmers talk about the DOM (Document Object Model) they are talking about the tree-like structure of a page

An element starts and ends with a tag. In its middle there can be content. This content can be text or another tag. Each tag can have attributes with values. CSS defines the style, meaning how elements look on the page. Every element is a box but with CSS you can change the form elements are displayed on the page. However, you can break down any webpage to their boxes. Boxes can be summarized with div-tags. Within these div-tags attributes can be determined that give the box a name.

Chapter 3: Webpages, Documents, and Structure

Adding CSS Style to HTML Structure

You can have several CSS (Cascading Style Sheets) files or sheets that will be referenced in HTML and all of them will be used to find out the final and visible style for the page. CSS is code written to control the "style" of HTML elements. Cascading means keep on going down with the rules and see which ones apply (priority rules). That idea is known as inheritance and understanding it requires abstract thinking.
In CSS you start with what is called a selector. It defines to what elements of a page a particular style applies to. A particular style can be expressed through a rule. A rule set consists of a selector followed by a declaration block. A declaration is made up of a property name and a value, separated by a colon; whitespace characters can appear around any of these elements {property: value}. If we want to apply a style to a bunch of grouped up html elements, we can use class selectors.

Avoiding repetition

Avoiding repetition is essential. When programmers don't avoid repetition, they will often have to do the same thing over and over.
This may mean copy and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times.
This can lead to errors when the programmer decides to make a change to something.
If they don't dilligently make that same change everywhere the repeated code appears, problems will arise.
